## How to Interpret Results

### Time Metrics
- **Time (ns)**: CPU time per operation in nanoseconds
- **Lower is better**: Smaller values indicate faster operations

### Speedup Factor
- **Nx faster**: Pool allocation time compared to standard allocator
- Example: "10x faster" means pool is 10 times faster

### Performance Thresholds

| Block Size | Expected Speedup | Status |
|------------|-----------------|--------|
| 64B | 10-50x faster | Target for small blocks |
| 256B | 5-20x faster | Good performance |
| 1KB | 2-10x faster | Moderate benefit |
| 4KB+ | 1-5x faster | Diminishing returns |
